## Why typed-xlsx?

Building Excel exports shouldn't be painful. typed-xlsx provides a type-safe, intuitive API for creating complex spreadsheets with minimal code.

## Simple Example

Here's a taste of what working with typed-xlsx looks like:

```typescript
import { ExcelSchemaBuilder, ExcelBuilder } from '@chronicstone/typed-xlsx'

interface User {
  id: number
  firstName: string
  lastName: string
  email: string
}

// Define your schema with full type-safety
const userSchema = ExcelSchemaBuilder
  .create<User>()
  .column('id', { key: 'id' })
  .column('firstName', { key: 'firstName' })
  .column('lastName', { key: 'lastName' })
  .column('email', { key: 'email' })
  .build()

// Build your Excel file
const buffer = ExcelBuilder
  .create()
  .sheet('Users')
  .addTable({
    data: users,
    schema: userSchema,
  })
  .build({ output: 'buffer' })
```

<Note>
  TypeScript will validate your column keys, transformations, and data structures at compile time, catching errors before they reach production.
</Note>
